延续早年对 CSS 样式与文件命名的讨论，2024–2025 年 BEM 与 Tailwind 仍是主流实践。本文简要对比并给出使用建议。

## 一、BEM：Block-Element-Modifier

BEM 由 Yandex 提出，被微信、JetBrains 等团队采用，适合组件化与多人协作。

### 三层结构

- **Block（块）**：独立可复用的组件，如 `.card`、`.button`
- **Element（元素）**：块内的子元素，用双下划线 `__` 连接，如 `.card__title`
- **Modifier（修饰符）**：块或元素的状态，用双连字符 `--` 连接，如 `.button--disabled`

### 命名约定

- 小写字母，单词间用 `-` 分隔
- 避免嵌套过深，保持选择器简洁

```css
/* Block */
.card { }

/* Element */
.card__header { }
.card__body { }

/* Modifier */
.card--featured { }
.card__header--collapsed { }
```

### BEM 的优势

- 选择器语义清晰，减少样式污染
- 不依赖 DOM 深度，便于重构
- 团队协作和代码 review 更直观

## 二、Tailwind：Utility-First

Tailwind 采用功能类优先，通过预定义类组合实现样式：

```html
<button class="px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600">
  按钮
</button>
```

### 实践建议

- 控制单元素上类名数量，必要时用 `@apply` 提取公共样式
- 善用响应式前缀（`sm:`、`md:`、`lg:`）做多端适配
- 在 `tailwind.config.js` 中定义设计令牌，保持风格统一

## 三、BEM 与 Tailwind 的取舍

| 场景 | 建议 |
|------|------|
| 组件库、多人协作 | BEM，结构清晰、易维护 |
| 快速原型、小项目 | Tailwind，开发效率高 |
| 混合项目 | 组件用 BEM，布局和简单样式用 Tailwind |

## 四、小结

- **BEM**：适合强调可读性、可维护性和团队协作的项目
- **Tailwind**：适合强调开发效率和一致性的项目
- 两者可并存：BEM 定义组件结构，Tailwind 处理布局与工具类

根据项目规模和团队习惯选择或组合使用即可。
